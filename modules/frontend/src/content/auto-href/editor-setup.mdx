TypeScript sees generated route types through SvelteKit's `icon="file" tsconfig.json`. Native suggestions inside Svelte markup use a second standard: VS Code-compatible HTML custom data.

## Configure the workspace

Run the init command from your application root:

		label="Deno"
		icon="deno"
		command="deno run -A npm:svelte-auto-href init"
	/>
	<CommandPicker.Item value="bun" label="Bun" icon="bun" command="bunx svelte-auto-href init" />
	<CommandPicker.Item

		command="npm exec -- svelte-auto-href init"
	/>
	<CommandPicker.Item
		value="yarn"

	/>

The command creates or updates `icon="vscode" .vscode/settings.json`:

```json name=".vscode/settings.json" icon="vscode"
{
	"html.customData": [".svelte-kit/svelte-auto-href/html-data.json"]
}
```

Existing JSONC comments, trailing commas, and unrelated settings are preserved. Running the command again is safe; the generated path is only added once.

The custom data supplies values for:

- `svelte <a href>` and `svelte <area href>`
- `svelte <form action>`
- `svelte <button formaction>` and `svelte <input formaction>`

Markup suggestions include static routes and concrete paths extracted from `ts entries()`. Dynamic patterns stay in TypeScript completions and the programmatic editor API, where inserting `/blog/${slug}` is meaningful code instead of literal HTML text.

## Configure an editor globally

Use global mode when you want every project to recognize the generated file without committing a workspace setting:

Supported editor values are:

- `vscode`
- `vscode-insiders`
- `cursor`
- `vscodium`
- `windsurf`

Global settings paths are resolved for Windows, macOS, and Linux. The default editor is `vscode`; pass `--editor` explicitly for a fork.

> [!TIP] Use workspace setup for teams
> Committing `icon="vscode" .vscode/settings.json` gives every contributor the same markup completions. Global setup is useful when you own all the projects on the machine or do not want an editor-specific file in the repository.

## When suggestions refresh

The Vite plugin generates metadata when the build starts and when the dev server starts. While Vite is running, adding, changing, moving, or deleting files and directories beneath the configured routes directory schedules another generation pass.

The plugin writes only when generated content changes, so normal route edits do not create unnecessary file churn.

If TypeScript suggestions are missing, confirm that Vite has run and that `icon="file" tsconfig.json` extends `icon="file" .svelte-kit/tsconfig.json`. If markup suggestions are missing, confirm that `html.customData` points to the generated `icon="file" html-data.json` file and restart the editor language server once.
