## Software around SER

### better-svelte-check

`better-svelte-check` loads your Vite config before it runs [svelte-check](https://svelte.dev/docs/cli/sv-check).
That gives SER a chance to lower its extended syntax, so the checker sees ordinary Svelte and TypeScript.

> [!WARNING] Not released yet
> `better-svelte-check` has not been released yet, so it is not currently available from the npm registry.

### svelte-plugin-composer

`svelte-plugin-composer` is a plugin for Vite that composes your plugins in the order you want them to run,
escaping plugins that use `ts enforce: "pre"` or `ts transform.order = "pre"`.

It is available for download via the npm registry.

## IDEs

For VS Code and its forks, there is an extension available for both
[Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=Barekey.svelte-effect-runtime-vscode)
and the [OpenVSX Registry](https://open-vsx.org/extension/barekey/svelte-effect-runtime-vscode) to enable IntelliSense for types and the extended grammar.

Both are available from the extensions tab in your code editor.

> [!TIP] About the registries
> Cursor and several other VS Code forks use [OpenVSX Registry](https://open-vsx.org) instead of Microsoft's [Visual Studio Marketplace](https://marketplace.visualstudio.com/).
> The extension works the same from either registry.
