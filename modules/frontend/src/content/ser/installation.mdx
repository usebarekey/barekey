SER only needs one package, but we recommend installing both of these:

- `svelte-effect-runtime`
- `svelte-plugin-composer`

Install both, then wire `ts effect()` through composer in `icon="vite" vite.config.ts`.

> [!NOTE] Why composer?
> While SER can run without `svelte-plugin-composer`, composer is the recommended way to use SER.
>
> It makes plugins in your Vite config run in the position you gave them instead of skipping the queue with `ts enforce: "pre"` or `ts transform.order = "pre"`. It also handles useful Svelte config work like deduping config contributions.

## Configure Vite

Update your `icon="vite" vite.config.ts` with the following code:

> [!IMPORTANT] Order matters
> Put `ts effect()` first in your plugin array. It transforms what other plugins might see as invalid syntax into valid syntax.

```ts diff name="vite.config.ts" icon="vite"
import { defineConfig } from "vite";
import { sveltekit } from "@sveltejs/kit/vite";
++ import { effect } from "svelte-effect-runtime";
++ import { compose } from "svelte-plugin-composer";

export default defineConfig({
++	plugins: compose([
++		effect(),
		sveltekit({
++			compilerOptions: {
++				experimental: {
++					async: true,
++				},
				runes: ({ filename }) =>
					filename.split(/[/\\]/).includes("node_modules") ? undefined : true,
			},
++			experimental: {
++				remoteFunctions: true,
++			},
		}),
++	]),
});
```

You should now have added `ts compose()`, `ts effect()` and enabled both experiments for remote functions and async rendering in Svelte.

> [!TIP] Remote functions is technically optional
> If you are not planning on using remote functions, you can safely disable them.
