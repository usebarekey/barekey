## What this error means and what can trigger it

`ts UnsupportedRemoteFormResponseError` means the response passed SER's top-level envelope check as `ts type: "result"`, but the result branch cannot supply a usable decoded payload. The class retains the original value in `ts envelope` and is nested in `ts RemoteTransportError.cause`.

Two shapes trigger it. The envelope may omit both of the supported encoded slots, `ts result` and `ts data`. Alternatively, one of those strings may parse with devalue but produce a value that fails SER's payload schema. That schema accepts an optional issue array and an optional result value; issue paths must consist of string or numeric segments.

Invalid devalue syntax is a different path. The parser throws before this explicit construction, and the outer Promise normalization handles that thrown parser error as a transport failure.

## Why this is the case

Recognizing the envelope's discriminator proves that the server intended a result response. It does not prove that the client can extract a result from it. Separating this case from `ts InvalidRemoteFormResponseError` pinpoints version skew and payload-contract changes after the top-level protocol has already succeeded.

SER validates the decoded value before reading issues or results from it. That check prevents arbitrary devalue output from being treated as validation data and gives the transport boundary one controlled failure shape.
