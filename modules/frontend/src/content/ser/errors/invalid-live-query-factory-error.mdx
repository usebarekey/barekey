## What this error means and what can trigger it

`InvalidLiveQueryFactoryError` covers three distinct live-query shape invariants.

The first invariant is simple: the generated native factory must be callable. Once called, it must produce a value with a callable `ts Symbol.asyncIterator`, which rules out ordinary query resources, arrays, primitives, and arbitrary objects. Reconnection adds one more requirement because `ts Live.reconnect(stream)` needs hidden live metadata whose native resource exposes a callable `ts reconnect` method.

One wrinkle is that the diagnostic message mentions an object with a `ts query` method, although the 4.0.0 client adapter accepts only a function at that boundary. The async-iterable validator and reconnect path inspect different shapes later on.

## Exactly where it triggers

Non-callable client factory, thrown synchronously during adapter creation:

```ts name="modules/svelte-effect-runtime/src/remote/client/query.ts" icon="ts" line="147" char="9" link="https://github.com/usebarekey/svelte-effect-runtime/blob/55b22b5afcfecf1a5643c5e26f7088fae6a22a6e/modules/svelte-effect-runtime/src/remote/client/query.ts#L147"
if (typeof native_factory !== "function") {
	throw new InvalidLiveQueryFactoryError();
}
```

Factory result without a callable async iterator, thrown synchronously by the live resource validator:

```ts name="modules/svelte-effect-runtime/src/live.ts" icon="ts" line="204" char="9" link="https://github.com/usebarekey/svelte-effect-runtime/blob/55b22b5afcfecf1a5643c5e26f7088fae6a22a6e/modules/svelte-effect-runtime/src/live.ts#L204"
if (!is_async_iterable_resource(resource)) {
	throw new InvalidLiveQueryFactoryError();
}
```

Missing reconnect metadata or method, returned as a lazy Effect failure:

```ts name="modules/svelte-effect-runtime/src/live.ts" icon="ts" line="188" char="23" link="https://github.com/usebarekey/svelte-effect-runtime/blob/55b22b5afcfecf1a5643c5e26f7088fae6a22a6e/modules/svelte-effect-runtime/src/live.ts#L188"

	return Effect.fail(new InvalidLiveQueryFactoryError()) as unknown as Effect.Effect<
		void,

}
```

The third path behaves differently: it is not a synchronous throw. It becomes a typed failure only when the reconnect Effect runs. For reference, the class itself is declared at `name="modules/svelte-effect-runtime/src/errors.ts" line="607" char="14" link="https://github.com/usebarekey/svelte-effect-runtime/blob/55b22b5afcfecf1a5643c5e26f7088fae6a22a6e/modules/svelte-effect-runtime/src/errors.ts#L607"`.

## Why this is the case

An Effect Stream can be built from a native async iterable only when that protocol is actually present. The explicit validator protects `ts Stream.fromAsyncIterable` from receiving a lookalike resource with unrelated methods.

Reconnect is stricter still. SER attaches hidden metadata only when it creates a remote live Stream, preserving the native resource needed for reconnection. A plain Stream cast to the remote type has no such resource, and a native resource without `ts reconnect` cannot perform the requested operation.
