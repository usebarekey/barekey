## What this error means and what can trigger it

`ts AwaitInEffectWorkError` means one script statement belongs to both JavaScript's Promise continuation model and SER's Effect generator model. The statement contains a top-level `ts yield*`, SER successfully lowered at least one Effect block from it, and the same statement also contains a top-level `ts await`.

Top-level is relative to the statement being inspected. The detectors do not descend through nested function boundaries. An ordinary statement such as `ts const response = await fetch(url)` does not trigger because it contains no SER work. A combined expression such as `ts record(await transform(yield* Load()))` does.

## Why this is the case

The lowered statement runs inside generator-based Effect sequencing, where `ts yield*` controls suspension, interruption, scope, and the typed error channel. Native `ts await` suspends an async JavaScript function around a Promise. SER would have to invent how Promise cancellation, rejection, and continuation order interact with the dispatcher-owned Effect fiber.

Rejecting the mixed statement keeps the two execution models explicit. Ordinary Svelte top-level await remains untouched in statements that SER does not lower.
