## What this error means and what can trigger it

`ts VitePreTransformPluginConflictError` describes Vite plugins that declare pre-transform priority and may inspect a Svelte file before SER lowers `ts <script effect>` or markup `ts yield*`. The class stores the candidate plugin names.

Current 4.0.0 production code never constructs, throws, or fails this class. It remains exported for compatibility. The only `ts new VitePreTransformPluginConflictError(...)` in the repository is its JSDoc example.

SER still detects the condition, but emits an informational logger notice. A plugin qualifies when it has a transform hook and either `ts enforce: "pre"` or an object-form hook with `ts transform.order: "pre"`. SER excludes its own plugins and known Vite/Svelte framework plugins from the candidate list.

## Exactly where it triggers

There is no production error trigger. The declaration is:

`icon="ts" modules/svelte-effect-runtime/src/errors.ts:85:14`

The current diagnostic path is the Vite plugin's `ts configResolved` hook, which checks the detected names and logs the notice instead of throwing:

`icon="ts" modules/svelte-effect-runtime/src/compiler.ts:149:4`

```ts name="modules/svelte-effect-runtime/src/compiler.ts" icon="ts"
const conflicting_plugin_names = find_pre_transform_plugin_names(config.plugins);

if (conflicting_plugin_names.length === 0) {
	return;
}

config.logger.info(make_pre_transform_plugin_notice(conflicting_plugin_names));
```

Consequently, no line and column can honestly be given for a runtime construction site. The source contains none.

## Why this is the case

Pre-transform ordering is evidence of possible competition, not proof of incompatibility. A listed plugin may ignore Svelte files, inspect unrelated syntax, or tolerate raw SER syntax. Stopping every build would turn a precaution into false failures.

The notice preserves visibility when another plugin reports parser errors before SER runs. The exported class preserves API compatibility for code that referenced the earlier structured diagnostic.
