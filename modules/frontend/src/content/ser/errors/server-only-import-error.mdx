## What this error means and what can trigger it

`ts ServerOnlyImportError` means a server-only export from the package root was invoked without SER's Vite rewrite replacing it with the implementation from `ts svelte-effect-runtime/server`. The error records the invoked export in `ts export_name`.

Function placeholders exist for `ts Query`, `ts Query.batch`, `ts Query.live`, `ts Command`, `ts Error`, `ts Form`, `ts Prerender`, `ts Redirect`, `ts RequestEvent`, and `ts get_server_runtime_or_throw`. `ts ServerRuntime.make` uses a class placeholder. Invocation can reach the placeholder when the SER Vite plugin is missing, a file does not follow recognized server naming conventions, or tooling executes the module without Vite's transform.

The rewrite recognizes `ts *.server.ts`, `ts *.remote.ts`, their module-suffix variants, and `ts hooks.server.ts`. It handles static imports, re-exports, and literal dynamic imports.

## Why this is the case

The root entrypoint must be importable by code that can enter a browser bundle. The real factories depend on SvelteKit server modules, request state, and server runtime services. Shipping those implementations through the universal root would weaken the server boundary.

The placeholders act as named tripwires. Correctly transformed server files never execute them; an untransformed invocation fails immediately with the exact export name instead of loading server code into an unsupported environment or producing a later undefined-function error.
