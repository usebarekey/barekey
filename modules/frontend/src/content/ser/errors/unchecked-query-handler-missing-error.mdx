## What this error means and what can trigger it

`ts UncheckedQueryHandlerMissingError` means `ts Query("unchecked", handler)` reached the server factory without a truthy second argument. The string sentinel disables input-schema validation; it does not contain the operation that reads data.

In untyped JavaScript or through an unsafe cast, omitted, `ts undefined`, `ts null`, `ts false`, `ts 0`, or an empty string as the second argument all reach this guard. The public TypeScript overload normally rejects these calls earlier. A truthy non-function does not hit this particular guard and can fail later while the wrapper is built or invoked.

## Why this is the case

The one-argument `ts Query(handler)` overload represents an inputless query. The two-argument unchecked overload uses its first argument only to select validation behavior. Without the guard, the factory could mistake the sentinel string for the inputless handler and pass it into the remote wrapper.

The explicit error keeps overload misuse separate from an eventual “value is not callable” failure and identifies that validation was intentionally disabled while execution logic was accidentally omitted.
